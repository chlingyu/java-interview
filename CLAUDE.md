# CLAUDE.md - 项目约束

## 八股文面试题写作风格规范（B+C 混合模式）

### 目标读者
基础较弱的 Java 开发者，准备 3~5 年经验的后端面试。

### 写作模式：B+C 混合

根据题目的面试高频程度，使用不同深度的写作风格：

#### ⭐⭐⭐⭐~⭐⭐⭐⭐⭐ 高频题 → C 风格（通俗故事型）

1. **用生活比喻解释原理**：先用一个通俗场景让读者理解概念，再过渡到技术细节
2. **专业术语首次出现时标注**：用 **加粗（括号解释）** 格式，例如：**哈希冲突（两个 key 算出了同一个位置）**
3. **关键数字用 ⚡ 标注**：所有阈值、默认值必须写死，不要模糊表述。例如：链表长度 ⚡**≥ 8** 且数组长度 ⚡**≥ 64** → 转红黑树
4. **复杂流程用分步骤 + 示意图**：拆成带编号的步骤，必要时生成图片辅助
5. **每题附"🎤 面试这样答"**：给出一段可以直接参考的面试回答话术
6. **适当配图**：数据结构图、流程图、架构图等

#### ⭐⭐~⭐⭐⭐ 普通题 → B 风格（结构化讲解型）

1. **一句话回答** + 关键原理简述
2. 保留专业术语标注
3. 核心代码示例（带注释）
4. 不需要比喻和"面试这样答"
5. 用表格对比差异，简洁明了

### 每道题的结构模板

#### 高频题（C 风格）模板

```markdown
### N. 面试题目？

> ⭐⭐⭐⭐⭐ 必考 | 难度：⭐⭐⭐

**一句话回答**：xxxxx

**通俗理解**：（用比喻/生活场景 + 专业术语标注解释原理）

**原理详解**：（分步骤 + 关键数字 ⚡ 标注 + 配图）

**代码示例**：（核心代码，带中文注释）

**🎤 面试这样答**：
> "xxxxxx"
```

#### 普通题（B 风格）模板

```markdown
### N. 面试题目？

> ⭐⭐ 了解 | 难度：⭐⭐

**回答**：简洁讲解原理，专业术语加粗标注。

**代码/表格**：（如需要）
```

### 注意事项

- **不要写干巴巴的纯表格堆砌**，高频题必须有温度
- **不要省略关键数字**，所有阈值、默认值都要写清楚
- **专业术语不能落下**，但第一次出现必须用通俗语言解释
- **代码示例要有中文注释**，不要写纯英文注释
- **篇幅控制**：高频题可以长，低频题要精简，每章控制在合理长度内
